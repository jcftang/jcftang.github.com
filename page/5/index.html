using git daemon to share your git repo on a trusted lan

02 Jul 2012, 00:00

One of the things that I don’t like about git is the interface at times can be too geared towards power users. One command that I particularly like in mercurial is the hg serve command, which is incredibly easy to use. To do the equivalent of the hg serve in git you can do the following… user$ cd my-repo user$ git daemon --export-all --base-path=$(pwd) The above assumes you are on a trusted network and you want to share your repo with people on your LAN and you are in the toplevel directory of the repository that you want to temporarily share.
